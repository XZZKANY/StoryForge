## 0. 北极星与不变量

- **飞轮愿景**：写 → 发 → 收集信号 → 喂 → 进化编辑器 → 写出更有风格的作品。读者信号不只调一本书，最终反哺编辑器本身。
- **红线**（不随阶段变化）：后端不写项目文件（proposed patch 确认制）；不伪造结果；发书如实勾 AI（AIGC 标识为法定义务）、不公开宣称 AI；质量竞争、不批量。
- **用户时间预算**：每周 ≤15 分钟拍板 + 一次 2-3h 真机波（Phase A 的 S10）；写作时间是创作不计入工程预算。
- **纪律**：probe 先于 build；QoL 由摩擦日志驱动（每周至多一刀）；别为未验证的方向重构；模型初稿永远留底（delta 标注数据）。

## Phase B：在编辑器上写作品（用户主导，我辅助）

- **B1（S3）手稿保险**（开写前夕建，不提前空转）：连载目录建在**仓库外**（位置用户定，拍板点 P3）→ `git init` + Windows 计划任务每 30 分钟自动 commit；编辑器 `.storyforge/versions` 为第二层保险。把创作资产从 `D:\记事本\StoryForge-n1连载-末世吞噬-创作资产存档-20260707.md` 搬入连载目录；`canon.json` 首刷吃末世系统数字状态（陈默 / 进化点 / 爪刃等——当时选题就是图状态可数字化、低漂移）。归属：我搭，用户定目录。
- **B2 n=1 接续**（不是从零轻启动）：Gate-0 Ch2（稿已在）→ Ch3 起草 → 黄金三章定稿 → 往后日更 / 隔日更节奏随写作状态定（无 deadline）。每章模型初稿与定稿并排留底。
- **B3 dogfood 摩擦环**：连载目录内固定 `frictions.md` 记摩擦；每周 ≤15 分钟分诊；每周至多一刀 QoL PR。编辑器功能打磨全走摩擦准入（D7 已拍板），不预先立项。

**Gate B→C**（用户拍板，拍板点 P2）：存稿 ≥8-10 章（建议值）+ 手稿保险在位。

## Phase C：发 + 收集信号（时点用户拍板）

- **C1 发番茄**：复用占位空号（改名改封面，不浪费月 3 本额度）；如实勾 AI；不公开宣称 AI。坑：别近 0 点发章（当日读完率 / 跟读率失真）。
- **C2 信号首采**（首秀 + 3 轮推荐验证窗口，约 10 万字前后）：带回三样——①章节跟读率曲线（对 §4 预注册预测表逐条对账）②流量构成（书城 / 分类算法推 vs 书架自来）③模型稿 → 定稿 delta 是否收敛（比跟读率更根本的隐藏指标）。
- **C3 复核承重假设**：「声明 AI 不限流」用流量构成拆分祛混淆（算法推趋零而书架有量 = 被限流非书差）。
