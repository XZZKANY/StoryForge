## 1. 我们到底在建什么

### 1.1 一句话

在写作 IDE 之上，建一套 **自用的** 番茄多笔名发行操作系统：  
创作产书 → 备发 → 开书 → 连载 → 止损/复盘；  
用多笔名身份池消化番茄开书配额。

### 1.2 双引擎（自用版）

```
创作引擎（StoryForge 主线） ──合格书稿──▶ 发行引擎（自用中枢）
                                              │
                                              │ 执行层见 §8 分级
                                              ▼
                                         番茄作者后台
```

对外可以长期不讲「发行双引擎」；对内它就是你的车队驾驶舱。

| 层 | 问题 | 主要靠什么解决 |

| L5 经营 | 养得住吗、何时止损 | 健康度 + 复盘 |

## 3. 战略阶段（自用节奏）

### Phase 0 — 总计划冻结（当前）
对齐目标、红线、番茄单平台、自用定位。  
**时间策略（已拍 C）**：本文档长期当宪法；工程节奏以「这季可跑 Phase 1 面板」为切片，不把 Phase 2–4 绑进同一交付。

### Phase 1 — 自用发行中枢（Desktop 内）= **管理面板**
完整经营闭环：Library、流水线、配额、日历、指派、Ready、作业包、今日作战、止损复盘。  
**弱耦合**：书从本地项目挂入即可管；不拦「没走开书向流水线」的书。  
**存活最小集（必带）**：同日开书错峰上限、风险号熔断停派、spare 产能显示（理论产能 − 预留 buffer）。  
**执行层（这季）**：**L0+L1** — 人在番茄点；SF 出物料、一键复制、检查清单；不含 L2 浏览器助手 / L3。  
**不对外主打、不包装成商业卖点。**

### Phase 2 — 创作侧供给对齐（可选增强，非面板前置）
开书向模板/Agent 流程、Ready 与质量门禁**可选咬合**、空位占坑逼稿。  
解决「有号没好书 / 开了就死」——但 **不推翻 Phase 1 弱耦合面板**。  
**2026-07-13 轻量切片已落地**：空位占坑/绑定、冷号限载、Ready 软门、简介去同质（见 `phase2-prd.md`）。Agent 开书向流水线仍可选后置。

### Phase 3 — 执行层增强（L0–L2，再评估 L3）
在 §8 分级内，降低「人肉点后台」成本：  
例如本机已登录浏览器辅助填表、半自动粘贴作业包字段、开书后回写向导。  
**L3（无人值守开书）**：自用可选，不默认排期；进入前单独拍板范围（是否含自动登录/定时提交）。  
**不含**：黑产养号（B）、产品内置反检测/打码（L4 不交付）。

### Phase 4 — 可选：自用 pack 化 / 多平台 / L3 落地
仅当你需要时：番茄规则外置；或其他平台；或把 L3 从「可选项」落成自用能力。  
仍可不对外。

## 4. 目标架构（番茄单平台）

```
Author
 ├─ Fanqie Identity Pool（笔名账号）
 ├─ Work Library（多书）
 ├─ Schedule / Assignment / Quota ledger
 ├─ Open Pack（上架物料）
 ├─ Ops signals（本地写作活跃度等）
 └─ Review

（platform 字段固定 fanqie 或仅支持 fanqie pack）
```

主程序接口仍保持干净，避免他日多平台时推翻；但 **实现与精力只投番茄**。

## 5. 与主线关系

- StoryForge 公开产品叙事仍可只打写作 IDE  
- 发行中枢 = **你的私有经营层**，挂在同一 Desktop  
- 不因自用车队去复活 Web 主入口或自动整书神话

## 6. 数据原则

- 本地优先；不存番茄密码到云  
- 项目内 publish 档案可随书备份  
- 配额账是 **经营镜像**；与后台不一致靠手动校准（除非未来有官方接口）  
- 预留占额度；已开额度不因止损退回

## 8. 自动化 / 养号 / 风控 — 边界（重要）

你问：「养号、绕风控、自动打平台后台是要用到的吧？」

**分开说，不能捆成一件事。**

### 8.1 真正会用到的（经营上需要）

| 需求（白话） | 是否需要 | 应用什么手段 |
| --- | --- | --- |
| 多个笔名/号轮流开书 | **要** | 身份池 + 排期 + 额度账 |
| 号「养得住」= 有更新、不秒死 | **要** | 连载健康、提醒、止损；**内容与更新纪律** |
| 少打字、少开错页 | **要** | 作业包、一键复制、检查清单、向导 |
| 降低重复点击 | **可选** | 本机辅助填表（你已登录的会话内） |
| 后台数据对齐 | **可选** | 手动校准；或官方 API（若有） |

这些是 **车队运营**，不是 **对抗平台安全体系**。

### 8.2 「养号」两个完全不同的含义

| 含义 | 态度 | 总计划位置 |
| --- | --- | --- |
| **A. 内容养成**：按平台规则正常更新、做人设与数据观察、冷号慢慢开书 | ✅ 需要 | L5 经营 + 日程提醒 + 复盘 |
| **B. 黑产养号**：批量注册、设备农场、模拟活跃、洗号卖号 | ❌ 不做 | 不进路线图 |

自用多笔名只需要 **A**。  
没有 B，一样可以跑 5–8 个自有号 × 15 本/月（取决于你是否合法持有这些号、以及供给质量）。

### 8.3 「自动打平台后台」分级（2026-07-13 修订：L3/L4 不再写死禁止）

| 级别 | 是什么 | 总计划 |
| --- | --- | --- |
| **L0 全手动** | 人在番茄点；SF 只出物料与台账 | Phase 1 基线 |
| **L1 辅助粘贴** | SF 生成字段；你一键复制到已打开的后台 | Phase 1–2 |
| **L2 本机会话助手** | 在你已登录的浏览器里，**你触发** 填表/打开页面（可见、可中断） | Phase 3 默认增强 |
| **L3 无人值守执行** | 在你已持有账号上：会话保持、定时开书/提交、回写状态（**拆项拍板，见 8.3.1**） | **自用可选**，不默认排进最近切片 |
| **L4 对抗检测能力** | 指纹伪造、关联清洗、打码平台、反检测代理链等 | **不进 StoryForge 产品交付**（无模块、无设计、无代码路径） |

**结论（修订后）：**  
- L0–L2：主路径，优先做。  
- L3：**不禁止**；自用车队可选项，单独立项与范围边界后再做。  
- L4：**不作为产品功能交付**；多号隔离与设备/支付策略由你 **线下运营纪律** 自行处理，不写进 SF 实现。

#### 8.3.1 L3 拆项（避免和「打码/对抗」捆死）

| 子项 | 说明 | 默认态度 |
| --- | --- | --- |
| L3-a 会话保持 + 定时提醒/半自动提交 | 已登录会话内按计划推进 | 可进路线图评估 |
| L3-b 自动登录与无人值守开书提交 | 定时真正点开书/上传 | 自用可选，风险自担 |
| L3-c 验证码/滑块自动过 | 打码与自动过检 | **不进产品**（与 L4 同类，不交付） |

> 拍板用语：可以说「允许 L3」，但实现范围默认指 **L3-a/b 自用自动化**，**不含 L3-c/L4**。

### 8.4 真目标：「别一封封一大堆」（存活 / 降连坐），不是「反检测代理链」

你要的是 **车队存活**：单号出事别拖死整池。这和「产品内置反检测代理链」不是同一题。

| 风险源 | 白话 | SF 能做什么 | SF 不做什么 |
| --- | --- | --- | --- |
| **行为连坐** | 同节奏批量开书/同模板同秒操作 | 排期打散、开书日错峰、智能指派限流、今日作战分批 | 伪造指纹/代理池 |
| **内容连坐** | 同简介/同开篇/同标签批量上 | 作业包差异检查、开书前相似度告警（可做） | 洗稿规避审核 |
| **更新连坐** | 全池秒死/全停更 | 连载健康、止损、负载看板 | — |
| **身份/环境关联** | 同设备/支付/主体被平台关联 | **只记账**：号侧「隔离备注/风险标签」（你自填） | 代理链、设备农场、关联清洗模块 |
| **额度/政策** | 超开、违规触达 | 配额账、预留、政策参数 | 骗平台额度 |

**产品侧「存活层」（进路线图，属 L5 + 调度增强，不是 L4）：**

1. **错峰与分散**：同日开书上限、同号冷却、笔名轮转、避免「一天 15 本齐射」  
2. **冷号/热号策略**：新号限开、观察窗后再加负载（内容养成 A）  
3. **爆仓熔断**：某号异常（你手动标「疑似风险/已限流」）→ 自动停派、改派他号  
4. **池子备份**：理论产能含 spare；目标 15 时建议 6–8 号，不把所有额度压满  
5. **物料去同质**：开书作业包生成后做「过近重复」提示  
6. **复盘**：止损率、单号封禁/限流事件台账 → 下月降负载  

**线下纪律（你自理，总计划只点名不交付实现）：**  
账号主体、设备、支付、网络如何隔离——属个人运营选择；**不在 SF 内实现反检测代理链**。

> 一句话：**保证不「整池陪葬」靠调度 + 养成 + 熔断 + 备份号；不靠产品内反检测。**

### 8.5 「绕风控 / L4」产品边界

- **产品内**：不设计、不实现反检测、打码、指纹/代理对抗模块。  
- **产品外**：环境隔离你线下自理；总计划不提供对抗方案。  
- 与「要存活」不矛盾：存活层（§8.4）做；对抗模块不做。

### 8.6 自用场景下，够不够用？

对「约 15 本/月、单人多笔名、番茄约 3 本/号」：

- 理论号数：≥5，建议 **6–8（含 spare）**，避免满载  
- **瓶颈通常仍是供给与更新**；连坐风险用 **错峰 + 熔断 + 备份** 压，不是代理链  
- L0+L1 可跑通；L2 省时间；L3 省人时但 **提高批量行为相似度风险** → 上 L3 必须绑 §8.4 限流/错峰  

### 8.7 总计划里的执行层表述（修订用语）

> 执行层优先做 **已合法持有账号** 上的 **减负、防错、可选 L3-a/b，以及存活层（错峰/熔断/养成/备份）**。  
> 不做黑产养号（B）。  
> **不交付** 反检测/打码/代理对抗（L4 与 L3-c）。
