# 写侧架构（StoryForge 落地方案，已验证可行）

## 结论：webview 内 raw fetch 方案跑通
2026-07-13 实测：在 muye 页面 context 用原生 fetch 发 new_article + publish_article：
- new_article 返回 code:0 —— 页面 fetch 被 secsdk 全局 hook，自动补 x-secsdk-csrf-token，**无需手动取 token**。
- publish_article 结构全对，仅因测试内容「大段落重复」被内容质检拒（code:-3026），换正常正文即可发布（用户手动那章已 code:0 成功）。
- 全程无需 a_bogus、无需 DOM 自动化、无需碰 ProseMirror 编辑器。

## 为什么读侧走 Rust、写侧走 webview
- 读接口无需 token → Rust cookie 代理直连（已接入代码）。
- 写接口需 x-secsdk-csrf-token；该 token 由 secsdk JS 每次生成、纯 HTTP 握手取不到（挑战返回空，已验证）。
- 但 token 只挡「页面外」请求；**页面自身 fetch 被 secsdk 全局 hook 自动带 token**。故写侧走「webview 内 fetch」。

## 写侧脱 webview（多号直连，2026-07-15）
webview 内 fetch 用共享会话，同一时刻只能发一个号。为支持多号并存，改为：
- **登录窗捕获 csrf**：登录 webview 的 initialization_script 钩 fetch/XHR.setRequestHeader，
  抓页面请求携带的 x-secsdk-csrf-token（secsdk 自动补的那个），经 `publish:csrf-captured` 回传，按号存 `csrfToken`。
  Cookie 到手后自动导航到 book-manage 触发 muye 发认证请求以捕获 token。
- **直连发章**：Rust reqwest 代理带「该号 Cookie + 该号 csrf 令牌 + Origin/Referer/UA」直发三步，不依赖 webview 登着谁。
  实测发现 csrf 可跨请求复用（见 README），故登录时抓一次即可复用。
- 有令牌走直连（按号隔离，多号可各自批量），无令牌回落 webview 内 fetch（旧路径保留）。
- **capability**：远程 webview 的 `__TAURI__.event.emit` 需 `core:event:allow-emit` + `remote.urls`，
  原配置漏授（cookie/回执 emit 真机会被 ACL 拒），已补 `publish-webview-capability`。
- 真机验证（令牌捕获/直连发章/多号并存）归 E2E-1。

## 推荐实现（Tauri）
1. 起隐藏 webview 加载 https://fanqienovel.com/main/writer/book-manage（cookie 已登录）。
2. 等应用初始化（~3-4s），注入 JS 执行发布逻辑，页面 fetch 自动补 csrf。
3. JS 用 window.__TAURI__ event 把结果回传 Rust。

## 注入 JS 核心逻辑（已实测，见 scratchpad/fq-apipublish.js）
两步，均 POST form-urlencoded，query 带 aid=2503&app_name=muye_novel：
1. new_article： body aid&app_name&book_id&need_reuse=1 → 拿 item_id
   （若响应无 item_id，查 /api/author/chapter/draft_list/v1?book_id=... 取最新草稿 item_id）
2. publish_article： body 见 章节发布.md 全字段（item_id, book_id, content(HTML), title,
   volume_id, volume_name, publish_status=1, timer_status=0, ...）→ code:0 进审核

## 内容注意
- 正文必须 HTML（<p>段落</p>，空行 <p><br></p>），≥1000 字，且不能大段重复（否则 -3026）。
- title 含「第N章」前缀；章节序号只支持阿拉伯数字。
- publish_status=1 立即发布；定时=timer_status=1 + timer_time 秒级时间戳。
