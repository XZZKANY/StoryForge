## 完整流程（3 步）
1. **new_article** (POST) — 建空草稿拿 item_id
   - body: `aid&app_name&book_id&need_reuse=1`
2. **cover_article** (POST) — 保存正文草稿（编辑时自动调；这才是存正文接口）
   - POST /api/author/article/cover_article/v0/
   - body: `aid, app_name, book_id, item_id, title, content(HTML), volume_name, volume_id`
   - 返回: `{code:0, data:{latest_version}}`
3. **publish_article** (POST) — 发布（字段见下）
   - 成功返回: `{code:0, data:{item_id, tips}}`

注：`edit_article` 是 **GET**（加载草稿，返回 media_id/title/content/volume_id/publish_status），不是存正文接口。

## publish_article body（实测全字段）
  aid, app_name, item_id, book_id,
  content = 正文 HTML（<p>段落</p>，空行 <p><br></p>）,
  title = 第N章 章节名,
  volume_id, volume_name = 第一卷：默认,
  publish_status = 1（发布）, timer_status = 0（0立即/1定时）, timer_time =（定时秒级时间戳）,
  need_pay = 0, device_platform = pc, speak_type = 0, use_ai = 1或2,
  timer_chapter_preview = [], has_chapter_ad = false, chapter_ad_types =

## 发布拦截（实测错误码）
- -3026 大段落重复（同一句反复）→ 内容不能大段重复
- -3010 本书中已有重复章节或段落 → 内容不能与本书已发章节重复
- -3009 提交频繁，请稍后再试 → 有频率限制，两次发布要间隔够
- 硬规则：标题≤30字（含「第N章」）、正文≥1000字、章节序号只阿拉伯数字
