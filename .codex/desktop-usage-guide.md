### 测试步骤

#### 1. 打开项目
- 点击左侧"打开项目"按钮
- 选择 `D:\test-storyforge-project`
- 应该显示 4 个 Markdown 文件

#### 2. 打开文件
- 点击任意 `.md` 文件（如 chapter-001.md）
- 编辑器自动展开在中间
```
┌──────────┬──────────────┬──────────────┐
│ 文件树   │   编辑器     │  Assistant   │
│ (250px)  │   (自适应)   │   (400px)    │
└──────────┴──────────────┴──────────────┘
```

#### 3. 编辑文件
- 在编辑器中修改内容
- 标题栏显示 `●` 未保存标记
- 按 `Ctrl+S` 保存

#### 4. 折叠面板
- 点击文件树右侧的 `◀` 按钮 → 折叠文件树
- 点击编辑器左侧的 `◀` 按钮 → 折叠编辑器
- 点击 Assistant 右侧的 `▶` 按钮 → 折叠 Assistant
- 折叠后显示展开按钮 `▶` 或 `◀`

#### 5. 拖拽调整宽度
- 鼠标移到面板边缘
- 光标变为 `↔`
- 拖动调整宽度
- 限制：文件树 200-400px，Assistant 300-600px

#### 6. 使用 AI Assistant
- 在底部输入框输入问题
- 点击快捷按钮：
  - 🔍 审阅这章
  - 👤 人物一致性
  - 📖 剧情连贯
- 按 `Ctrl+Enter` 发送
- （当前显示模拟响应，后续接入真实 API）

## 🔧 常见问题

### Q: 窗口没有打开？
**A**: 检查：
1. 前端是否运行：`curl http://localhost:3007`
2. 进程是否存在：`tasklist | findstr storyforge`
3. 查看日志是否有错误

### Q: 编辑器没有显示？
**A**: 默认只显示文件树 + Assistant，点击文件后编辑器自动展开

### Q: 无法保存文件？
**A**: 确保：
1. 文件不是只读
2. 有写入权限
3. 路径正确

### Q: AI 没有响应？
**A**: 当前是模拟响应，后续会接入真实 API

### Q: 如何停止应用？
**A**:
- 关闭窗口
- 或按 `Ctrl+C`（终端）
- 所有服务会自动清理
