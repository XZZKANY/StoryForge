### 2. 测试 Blueprint 创建向导

**路径：** http://localhost:3000/blueprints

**操作流程：**

1. 点击右上角「新建 Blueprint」按钮
2. **步骤 1 - 故事立意：**
   - 输入前提：`林岚在雾港追查失真的灯塔信号，发现背后是一场跨越三十年的记忆实验。`
   - 选择基调：`克制悬疑`
   - 点击「下一步」

3. **步骤 2 - 规模配置：**
   - 目标章节数：`6`
   - 目标总字数：`9000`
   - 单章最小字数：`1000`
   - 单章最大字数：`1800`
   - 点击「下一步」

4. **步骤 3 - 高级选项（可跳过）：**
   - 视角角色：`林岚`
   - 主要场景：`雾港`
   - 点击「下一步」

5. **步骤 4 - 确认创建：**
   - 检查配置摘要
   - 点击「创建 Blueprint」

**预期效果：**
- ✅ 进度指示器显示当前步骤
- ✅ 表单验证工作（试试输入 2 章，会提示错误）
- ✅ 提交后看到右下角 Toast 通知："创建成功"
- ✅ 模态框关闭，页面刷新显示新 Blueprint

## 已知问题与解决方案

### 问题 1: API 服务未启动

**现象：**
- 点击「锁定 Blueprint」后 Toast 显示："操作失败: 500"
- 控制台显示 `fetch` 错误

**解决方案：**
```bash
# 手动启动 API（在新终端）
cd D:/StoryForge/apps/api
uv run uvicorn app.main:app --reload --port 8000
```

### 问题 2: 类型错误（开发环境）

**现象：**
- TypeScript 编译警告
- 测试文件类型不匹配

**影响：**
- 不影响运行，仅影响开发体验

**解决方案：**
- 按照报告中的说明，在测试 mock 数据中添加缺失字段

### 问题 3: 端口冲突

**现象：**
- `pnpm dev` 提示 "Port 3000 is already in use"

**解决方案：**
```bash
# 找到占用进程并结束
netstat -ano | grep :3000
taskkill /PID <进程ID> /F

# 或使用其他端口
PORT=3001 pnpm dev
```
