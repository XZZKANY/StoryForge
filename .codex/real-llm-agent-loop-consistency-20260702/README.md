# Agent loop 循环内 project.consistency 真·LLM 实跑证据（2026-07-02）

## 场景

环境同前两轮（run_windows.py 单机 sqlite 起服 + `STORYFORGE_LLM_CONFIG_FILE` 挂 deepseek-v4-flash BYO-key，真实 WebSocket）。验证本轮新增的项目级一致性观察工具。

单条消息：「帮我检查全书的一致性：主要人物的称谓在各文件里是否统一、时间线标记有没有可疑跳跃、有没有重复表达。先看设定确定人物，再用一致性扫描。」

## 结果

- **自主工具链**：5 轮 / 7 次工具调用，27.8s——fs.list → fs.read×3（先读设定 / 大纲确定人物）→ **project.consistency**（词条分布 + 时间标记 + 重复子句一次拿全）→ fs.read×2（按系统提示回原文抽查核实）。
- **结论质量（人工核对全部属实）**：
  - 称谓：正确识别「裴少卿」（第01章小吏口中）与「裴砚」（第02章叙述）是同一人的合理称谓分工，判为「叙述用名、对话用官职」而非误报冲突；观澜先生正文未出场与大纲第三幕安排一致。
  - 时间线：判定第 01→02 章为同一夜连续场景、大纲两处「十七年前」指向同一旧案，无矛盾——与埋设事实一致。
  - 重复表达：扫描为空，如实报告「无」，未伪造问题。
  - 增值观察：口头禅「星不欺人」仅出现一次（机械分布信号 → 模型转化为创作建议）。
- **证据链**：`assistant_tool_calls` 含 project.consistency 逐调用记录（input terms / output_summary scanned_files=5）；事件渐进到达。
- 脱敏扫描确认证据目录不含 API key。

## 判定与边界

- 通过：一致性观察工具在真实 provider 下被自主选用、信号被正确消化（含"合理区分不算冲突"的语义判断——这正是机械信号 + LLM 推理分工的设计意图）。
- 不外推：单一 provider；测试项目仅 5 文件（大项目的截断路径未实跑，有单测覆盖）；工具只报观察不下结论，结论质量取决于模型；不构成长程质量结论。
