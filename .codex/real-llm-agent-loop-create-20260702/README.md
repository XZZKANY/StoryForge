# Agent loop 循环内 file.create 新文件起草真·LLM 实跑证据（2026-07-02）

## 场景

环境同前几轮（run_windows.py 单机 sqlite 起服 + `STORYFORGE_LLM_CONFIG_FILE` 挂 deepseek-v4-flash BYO-key，真实 WebSocket）。验证本轮新增的循环内新文件起草工具（写作任务循环化）。

单条消息：「帮我写第三章：观澜先生在卦摊上第一次正面出场，与沈青梧有一段试探性对话，结尾埋一个与铜镜凿星有关的钩子。先看大纲、设定和前两章，然后起草到合适的新文件里，给我待确认补丁。」

## 结果

- **自主工具链**：8 轮 / 12 次工具调用，52.1s——fs.list → fs.read×5（大纲 / 设定 / 前两章）→ fs.search×2（核对线索衔接）→ **file.create**（起草 `正文/第03章.md`，488 字）→ **project.consistency**（起草后自主复核称谓 / 时间线，无提示要求）。
- **补丁契约**：`kind=file_revision` + `created_by_tool=file.create`、`before=""`、`file_path` 为项目内绝对路径、`requires_confirmation=true`；run 暂停等作者确认。
- **写回红线实证（双重）**：跑完盘上 `正文/第03章.md` 不存在（补丁未确认不落盘）；且模型两次尝试 fs.read 该未确认文件均被正确拒绝「文件不存在」——工具面与补丁面一致，模型无法把未确认草稿当已落盘事实。
- **草稿质量（人工核对）**：守住大纲约束（观澜「前任灵台郎」身份未提前揭晓，符合第三幕安排）；未泄漏「扳指 / 十七年 / 钦天监旧案」等后幕线索；结尾埋「天枢」钩子与铜镜凿摇光呼应；回话明确「确认后才会写入」。
- 脱敏扫描确认证据目录不含 API key。

## 判定与边界

- 通过：写作任务（新文件起草）在真实 provider 下经工具循环成立，读上下文 → 起草 → 自主一致性复核 → 待确认补丁全链自主决策。
- 不外推：单一 provider；真机 GUI 上「自动打开新文件 + PatchReviewPanel 确认写回」观感未验（本轮前端改动只有单测 + 浏览器 smoke 证据）；一次对话一个补丁为设计约束。
