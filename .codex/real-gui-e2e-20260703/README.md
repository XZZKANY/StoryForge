## 发现的问题

1. **P1 起草内容首行截断**：第03章补丁 `after` 首行只剩「渡」（应为「# 第三章 晨渡」）。数据库 `agent_artifacts` 里存的补丁即已截断，GUI 展示与写盘均如实传递——问题在后端 `file.create → draft_file_content` 的 LLM 产物路径。高度疑似 `strip_reasoning_leak`（app/common/llm_http.py）在模型把 `</think>` 关闭标签吐在标题中间时把「…# 第三章 晨」连同 think 块一起剥掉；原始响应未落盘，无法终审归因。建议：起草/修订路径记录原始响应 + 对产物做首行结构合理性校验。
2. **修订补丁 EOL 噪音**：一词之改（右手→左手）统计为 +19/-20，diff 面板整章标红绿。模型在回复里自行提醒「可能是换行符差异导致的噪音」。写回后内容无损，但 diff 可读性被破坏。建议补丁生成前统一换行符。
3. **聊天消息 markdown 不渲染**：`##`、`**`、表格管道符全部按字面显示（06/08），长回答观感明显受损。
4. **会话切换后运行面板串台/状态过期**（25）：切回会话 1 时底部流程树显示的是会话 2 的 file.create 运行，且状态卡仍是「暂停/等待权限确认/待确认补丁 #4」——该补丁此前已接受写盘。若运行面板按项目级设计，至少状态应刷新为已完成。
5. **完成态计数不闭合**：修订运行已标「已完成」但计数 7/9，`permission.confirm` 步骤图标保持进行中空心态（18 左侧）。
6. **发送快捷键不一致**：欢迎页 composer 纯 Enter 发送，聊天 composer 需 Ctrl+Enter（placeholder 未提示），首次使用会以为没反应。
7. 聊天 composer 无方向键输入历史（方向键能力仅在命令面板）；dev 模式 Monaco 报 web worker 回落警告（打包版待复核）。
